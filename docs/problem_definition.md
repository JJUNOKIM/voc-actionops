# VOC ActionOps 문제 정의

## 1. 프로젝트 개요

VOC ActionOps는 고객 리뷰, 문의, 설문, 앱 리뷰 등 여러 채널에 흩어진 고객 피드백을 수집하고, 반복적으로 발생하는 문제를 이슈 단위로 묶은 뒤, 우선순위 산정과 담당자 액션 관리, 처리 후 지표 추적까지 제공하는 AI 기반 고객 피드백 운영 플랫폼이다.

이 프로젝트는 단순히 고객 피드백을 감성분석하거나 요약하는 데 목적이 있지 않다. 핵심 목적은 고객 피드백이 실제 개선 작업으로 이어지도록 만드는 것이다.

## 2. 배경

기업은 다양한 채널을 통해 고객 피드백을 수집한다.

* 상품 리뷰
* 앱 리뷰
* 고객센터 문의
* 설문 응답
* 내부 테스트 피드백
* 소셜 채널 반응

하지만 피드백이 여러 채널에 분산되어 있으면 반복적으로 발생하는 문제를 빠르게 파악하기 어렵다. 또한 개별 피드백의 긍정·부정 여부만 확인해서는 어떤 문제를 먼저 해결해야 하는지 판단하기 어렵다.

실제 운영 환경에서는 다음과 같은 질문이 더 중요하다.

* 어떤 문제가 반복적으로 발생하고 있는가?
* 이 문제는 최근에 급증했는가?
* 부정 감성이 강한가?
* 긴급하게 대응해야 하는 문제인가?
* 담당자는 누구인가?
* 해결 이후 관련 불만이 줄었는가?
* AI가 분류한 결과를 사람이 검토하고 수정할 수 있는가?

VOC ActionOps는 이러한 질문에 답하기 위해 설계한다.

## 3. 기존 방식의 문제

### 3.1 피드백이 여러 채널에 분산됨

고객 피드백은 리뷰, 문의, 설문, 앱 리뷰 등 여러 출처에 흩어져 있다. 각 채널에서 개별 데이터를 확인할 수는 있지만, 여러 채널을 통합해 반복 이슈를 파악하기 어렵다.

### 3.2 단순 감성분석만으로는 우선순위를 판단하기 어려움

긍정, 부정, 중립 분석만으로는 실제 운영 우선순위를 정하기 어렵다. 예를 들어 부정 피드백이 많더라도 최근 증가율이 낮거나 고객 영향도가 낮을 수 있다. 반대로 피드백 수는 적지만 결제 실패, 로그인 장애처럼 긴급도가 높은 문제가 있을 수 있다.

### 3.3 분석 결과가 액션으로 이어지지 않음

대시보드에서 피드백 통계를 확인하더라도, 해당 이슈를 누가 담당하고 어떤 상태로 처리 중인지 관리하지 않으면 실제 개선 작업으로 연결되기 어렵다.

### 3.4 AI 분석 결과를 검증하기 어려움

AI가 피드백을 자동으로 분류하거나 요약하더라도 결과가 항상 정확하지는 않다. 따라서 원문 근거, 신뢰도, 수동 수정 이력, 수정 사유를 함께 관리해야 한다.

### 3.5 해결 후 개선 효과를 추적하기 어려움

이슈가 해결되었다고 하더라도, 관련 부정 피드백이 실제로 감소했는지 확인하지 않으면 개선 효과를 판단하기 어렵다.

## 4. 해결 방향

VOC ActionOps는 다음 흐름으로 문제를 해결한다.

1. 여러 채널의 고객 피드백을 표준 스키마로 수집한다.
2. AI 분석을 통해 감성, 카테고리, 긴급도, 요약 정보를 생성한다.
3. 유사한 피드백을 이슈 단위로 클러스터링한다.
4. 빈도, 부정도, 긴급도, 최근 증가율, 고객 영향도를 기준으로 우선순위를 계산한다.
5. 이슈별 담당자와 액션 아이템을 관리한다.
6. 처리 상태를 추적한다.
7. 해결 이후 관련 피드백 변화와 재발 여부를 모니터링한다.
8. AI 분석 결과에 대한 수동 수정 이력을 저장한다.

## 5. 핵심 가치

### 5.1 피드백을 이슈 단위로 재구성

개별 피드백을 단순히 나열하지 않고, 반복적으로 나타나는 문제를 하나의 이슈로 묶어 운영자가 문제 단위로 대응할 수 있게 한다.

### 5.2 AI 결과의 검증 가능성 확보

AI 분석 결과에 원문 근거, 신뢰도, 수정 이력을 함께 저장한다. 이를 통해 AI 결과를 그대로 신뢰하는 것이 아니라 사람이 검토하고 보정할 수 있는 구조를 제공한다.

### 5.3 운영 우선순위 제공

단순 빈도순이 아니라 부정도, 긴급도, 최근 증가율, 고객 영향도를 함께 반영해 우선순위를 계산한다.

### 5.4 액션 관리

이슈를 담당자에게 배정하고, 상태 변경, 코멘트, 액션 아이템을 통해 실제 개선 작업으로 연결한다.

### 5.5 처리 후 효과 추적

이슈 해결 이후 관련 피드백 수, 부정 피드백 비율, 재발 여부를 추적해 개선 효과를 확인한다.

## 6. 프로젝트의 핵심 메시지

VOC ActionOps는 고객 피드백을 단순히 분석하는 서비스가 아니라, 고객 피드백을 이슈로 구조화하고 실제 개선 액션까지 연결하는 운영 플랫폼이다.

AI는 최종 판단자가 아니라 피드백 분류, 요약, 이슈 후보 생성, 우선순위 산정을 보조하는 역할로 사용한다. 최종 운영 판단은 원문 근거와 수정 이력을 바탕으로 사람이 검토할 수 있게 설계한다.
